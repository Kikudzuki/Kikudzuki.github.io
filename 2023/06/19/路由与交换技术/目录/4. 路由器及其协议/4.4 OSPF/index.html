1. OSPF简述

OSPF（Open Shortest Path First，开放最短路径优先）是IETF 开发的基于链路状态的自治系统内部路由协议，仅传播对端设备不具备的路由信息，网络收敛迅速，有效避免了网络资源浪费。OSPF直接工作于IP层之上，IP协议号为89，并且以组播地址发送协议包。

2. OSPF优点

  1. 开销减少
    • OSPF使用组播发送路由信息，并且只在拓扑结构发生变化时发送路由表变化的更新信息。
  2. 支持VLSM和CIDR
    • OSPF路由更新中包含子网掩码。
  3. 支持不连续网络
  4. 支持手动路由汇总
  5. 收敛时间短
  6. 无环拓扑产生

3. Router-ID

RID默认为路由器最大的IP地址号，也可以手动设置为任意一个32bit的无符号整数，是一台路由器的唯一标识，在整个自治系统内唯一。

4. 工作原理

  1. OSPF协议的度量值

OSPF协议使用最短路径优先算法（SPF）来计算生成路由表，其参考值为链路成本（Cost）。链路成本的定义是基于链路带宽的，带宽越高则链路成本越低，带宽越低则链路成本越高。

  1. OSPF区域
    • 骨干区域：区域0为骨干网，其他网络要与骨干网紧密相连。
    • 非骨干区域：即非区域0。非骨干区域的必须要与骨干区域相连接，物理位置上不连接需要用虚连接（Virtual-link）连接到骨干区域。
    • OSPF区域外的路由器无法看到区域内的网络拓扑情况，这样不仅降低了网络流量，而且把SPF算法限制在一个比较小的范围中。而且一个区域的拓扑结构变化，也不会导致全网的OSPF重新计算路由。
  2. 标准OSPF区域
    • 特点
      区域内包含一组连续的节点和网络
      每个区域内路由器都有该区域的拓扑结构数据库，并运行着同一个SPF算法
      每个区域都与骨干区域紧密相连
      可以使用虚连接连接物理位置上与骨干区域不能直接相连的区域
    • 设计规则
      骨干区域必须最先存在
      每个区域与骨干区域紧密相连
      骨干区域必须是连续的
      非骨干区域可以使用虚连接和骨干网相连（临时手段）
  3. OSPF区域类型
    • 骨干区域（区域0）
      • 骨干区域是OSPF自治系统的核心，负责互连多个非骨干OSPF区域。OSPF骨干区域具有标准区域的所有属性，可以在物理位置上不连续，但在逻辑上必须连接（可以使用虚连接实现）。
    • 末节区域（Stub区域）
    • 完全末节区域（Totally Stub区域）
    • 次末节区域（NSSA区域）
  4. OSPF路由器类型
    • 区域内部路由器(IR)
    • 区域边界路由器(ABR)
    • 自治系统边界路由器(ASBR)
    • 骨干路由器(BR)

6. 常用命令的检测和管理命令

命令 作用
show ip ospf 显示OSPF进程和一些细节，如进程ID和路由器ID
show ip ospf database 显示路由器的拓扑数据库
show ip ospf neighbor 显示OSPF邻居
show ip ospf neighbor detail 显示OSPF邻居的细节信息，提供一些参数如邻居地址、Hello间隔、Dead间隔
show ip ospf interface 对一个给定的接口，显示OSPF的配置信息
ip ospf priority 接口命令，用于改变DR/BDR的选择过程
ip ospf cost 接口命令，用于改变一个OSPF接口的成本
